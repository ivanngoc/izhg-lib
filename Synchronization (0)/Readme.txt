Создает точку синхронизации в момент исполнения которой все методы будут выполнены по заданным условиям в заданном порядке.

Практическое применеие: при архитектуре системы обновления в виде слоенного пирога бывает нужно обеспечить выполнение после например второго и пятого слоя некотрую логику.
Также применеие может стоят в том что существует некоторый процесс раскиданный по слоям данные которого нужно сохранять и передавать их слоя в слой. По умолчанию без синхронизации можно просто передавать данные от ранних слоев к поздним. Но все же требуется декларация некоторых принципов и правил

может также применятся для сопряжения разных периодов обновлений. Например часть данных расчитывается каждый кадр, часть каждую секунду а часть каждые 4 кадра. Необходимо по общему знаменателю то есть по окончанию самого длительного цикла выполнять некоторую операцию. В данном примере наприме самое долгое это 1 секунда поэтому по факту каждую секунду будет выполняться этот процесс. Но также может выполняться и по самому короткому. Здес как будет отконфигурировано. Наприме если поворот меняется раз в 1 секунду а позиция присваиватеся каждый кадр то для плавности лучше выполнять процесс каждый кадр если это позволительно.
