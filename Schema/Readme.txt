Предоставляет синтаксис для проектирования системы.
Цель использования: упорядочивание выполнения по типу.
Например: небходимо установить четкий порядок выполнения объектов UserAction. 
Объекты еще не созданы, поэтому правила порядка определяются по типу. After<T>()/Before<T>()
Создается объект схемы по которому во время выполнения выдергиваются объекты заданного типа по порядку определенному в схеме.
Данный способ работает только при условии синглтонов или использования лишь по 1 объекту конкретных типов на схему.
